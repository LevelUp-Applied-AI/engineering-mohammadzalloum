# Pre-PR Self-Review Checklist

- [ ] Title is specific and uses imperative mood (e.g., "Fix …", "Add …")
- [ ] PR scope is one logical change (no "and" in the title)
- [ ] Code does what the title claims and handles main edge cases
- [ ] Tests are added/updated for the change, and all tests pass locally
- [ ] No debug artifacts left (print, breakpoint, commented code, TODOs)
- [ ] README/docs updated if usage/behavior changed
- [ ] I reviewed the diff myself for accidental changes (formatting, unrelated files)
- [ ] New dependencies are justified and minimal
